# Uplift Modeling(增益建模 / 个体处理效应)原理与适用场景

## 一句话概括

Uplift Modeling 估计的不是"处理(treatment)平均带来多少效果",而是**每个个体(或子群)被处理后结果会变化多少**——即条件平均处理效应 CATE $\tau(x)=E[Y(1)-Y(0)\mid X=x]$,目的是在预算有限时找出"被处理才会改变行为的那群人(persuadables)",把资源精准投给他们。

## 适用场景

**适合用 Uplift 的业务问题:**

- **营销/留存的"该对谁投放"**:发券、推送、外呼、广告的预算有限,想挑出"投了才会转化、不投就不会转化"的**被打动者(persuadables)**,而不是把券发给本来就会买的人。
- **关注个体异质效应而非总体平均**:同一个活动对不同人群效果可能完全相反(有人被激活、有人反而被打扰退订),需要按人排序、分层施策。
- **有实验或可去混淆的数据 + 想做排序决策**:手上有 A/B test(或可校正混淆的观测数据),既有处理标记又有结果,目标是给每个人打一个"增益分"并据此 targeting。
- **典型场景**:挽留流失用户(只挽留"会被挽留动"的)、交叉销售名单优选、广告受众筛选、推送频控(避免打扰会反感的人)。

**不适合用 Uplift 的场景:**

- **只关心整体平均效应(ATE/ATT)**:只想知道"这次活动整体增量多少",用 PSM / IPW / AIPW / DID 即可,无需逐人建模。
- **没有处理变异**:全部人都被处理、或全部都没被处理,没有对照,无法估计差异。
- **样本太小或结果太稀疏**:CATE 是两个预测之差,本身方差大,小样本下噪声会淹没真实信号,排序不可靠。
- **观测数据且无法去混淆**:若处理是按规则/人工挑选且关键混淆不可观测,直接建 CATE 会把选择偏差当成处理效应。

## 核心原理

### CATE / ITE 的定义

用潜在结果框架:个体 $i$ 被处理时的结果记 $Y_i(1)$,不被处理时记 $Y_i(0)$。**个体处理效应 ITE** 为 $Y_i(1)-Y_i(0)$。但同一个个体在同一时刻只能观测到其中一个(另一个是反事实,永远缺失),所以 ITE 本身不可直接观测。

于是退而求其次,估计给定协变量 $X$ 的**条件平均处理效应 CATE**:

$$\tau(x)=E[Y(1)-Y(0)\mid X=x]$$

即"长得像 $x$ 这类人,处理平均带来多少结果变化"。Uplift 模型输出的 $\hat\tau(x)$ 就是每个人的"增益分",用来排序和 targeting。

### 四象限人群

按"被处理是否转化"和"不被处理是否转化"的组合,把人分成四类,这是 uplift 决策的核心解读:

| 类型 | 不处理 | 处理 | uplift | 该不该投 |
|------|--------|------|--------|----------|
| **Persuadables(被打动者)** | 不转化 | 转化 | 正,且大 | **该投**,这是唯一真正创造增量的人群 |
| **Sure Things(铁杆)** | 转化 | 转化 | ≈0 | 不必投,投了也是白花钱 |
| **Lost Causes(无望者)** | 不转化 | 不转化 | ≈0 | 不必投,怎么投都不转化 |
| **Sleeping Dogs(睡狗 / 反感者)** | 转化 | 不转化 | **负** | **千万别投**,投了反而把人推走 |

普通的"响应模型/转化率预测"会把 sure things(本来就会买)排在最前面,导致预算浪费;而 uplift 模型专门把 persuadables 排前、sleeping dogs 排后。

### Meta-learners(元学习器)

把 CATE 估计转化为若干个普通监督学习问题,可复用任意 sklearn 回归/分类器:

- **S-learner(Single)**:训**一个**模型,把处理标记 $T$ 当成一个普通特征,拟合 $\hat\mu(x,t)=E[Y\mid X=x,T=t]$。预测时分别代入 $T=1$ 和 $T=0$ 求差:$\hat\tau(x)=\hat\mu(x,1)-\hat\mu(x,0)$。最简单,但当处理效应较弱时,模型可能"忽略" $T$ 这一维,把 uplift 压向 0。
- **T-learner(Two)**:对处理组和对照组**各训一个**模型 $\hat\mu_1(x)$、$\hat\mu_0(x)$,uplift 为两者之差 $\hat\tau(x)=\hat\mu_1(x)-\hat\mu_0(x)$。更灵活,能让两组各自拟合;但当某一组样本很少时该组模型方差大,两个模型的误差不会相互抵消,uplift 噪声放大。
- **X-learner**:在 T-learner 基础上,用一组模型为另一组个体**插补 ITE**,再分别拟合两组的 CATE,最后用倾向得分加权融合。**适合处理组/对照组严重不平衡**(如处理组很小)的场景,通常比 T-learner 更稳。

### Uplift Tree(增益树)简述

普通决策树按结果纯度(基尼/熵)分裂;**uplift tree** 改成按"分裂后两个子节点中处理组与对照组的响应差异最大化"来选分裂点(常用分布散度准则,如 KL、欧氏距离、卡方)。叶子节点直接给出该子群的 uplift 估计,天然产出可解释的 targeting 分群。随机森林版本即 uplift forest / causal forest。

### 评估:Qini 曲线 / uplift 曲线 / AUUC

由于 ITE 不可观测,**不能**用普通分类/回归指标逐人对错评估,只能评估**排序能力**——把模型分高的人优先处理,能否带来更多累积增量。做法:

1. 按预测 uplift **降序**排序全体样本。
2. 沿排序累积,在每个比例处计算"已处理人群中,treated 的累积响应 减去 control 的累积响应(按两组样本量校正)"。
3. 画出"处理人数比例 → 累积增量响应"曲线:
   - **uplift 曲线**:纵轴为累积增量;
   - **Qini 曲线**:Qini 把对照组响应按两组规模重新标定后相减,纵轴为累积增量人数,更适合不平衡场景。
4. 曲线越往左上凸、离"随机投放对角线"越远,说明排序越好。用**曲线下面积**汇总:**AUUC(Area Under the Uplift Curve)** 或 **Qini 系数**(模型曲线与随机对角线之间的面积,常做归一化)。

**为什么不能用普通分类指标**:一个模型可能把"谁会转化"($Y$)预测得非常准(高 AUC),但它排的最前面全是 sure things(本来就转化、uplift≈0),targeting 价值为零。AUC/RMSE 衡量的是对结果 $Y$ 的拟合,而 uplift 关心的是对**效应 $\tau$** 的排序,两者目标不同。

## 关键假设

Uplift 给出的是因果排序,依赖以下假设;被破坏时 CATE 有偏。

1. **随机化 / 可忽略性(Unconfoundedness)**:
   最好处理是**随机分配**(A/B test),此时 $(Y_0,Y_1)\perp T$,treated 与 control 的差异直接无偏。
   若为观测数据,需满足在控制协变量后 $(Y_0,Y_1)\perp T\mid X$——所有同时影响"是否被处理"和"结果"的因素都已测量。
   **违反后果**:存在未观测混淆时,模型把选择偏差当成处理效应,uplift 排序失真。

2. **正性 / 共同支撑(Positivity / Overlap)**:
   在协变量任意取值处,被处理和不被处理的概率都严格介于 0 和 1 之间:$0<e(x)<1$。
   **违反后果**:某些子群只有处理组或只有对照组,该处的 uplift 只能靠外推,不可信。

3. **SUTVA**:
   个体潜在结果只取决于自己是否被处理,无干扰/溢出,且处理只有单一版本。
   **违反后果**:存在网络溢出或多版本处理时,潜在结果定义被破坏,效应失去意义。

**观测数据务必先去混淆**:若不是随机实验,应先用 PSM/IPW 校正,或用 AIPW 等构造**去偏的伪结果**,再在其上建 uplift,否则 CATE 从一开始就是偏的。

## 与其它方法的关系

| 方法 | 校正什么 | 估计的量 | 用途 |
|------|----------|----------|------|
| **PSM** | 仅可观测混淆(配对) | 平均效应(ATT/ATE) | 整体增量评估 |
| **IPW** | 仅可观测混淆(加权) | 平均效应(常用 ATE) | 整体增量评估 |
| **AIPW(双稳健)** | 可观测混淆 + 结果模型 | 平均效应,双稳健 | 整体增量评估 |
| **DID(双重差分)** | 可吸收不随时间变的不可观测混淆 | 平均效应 | 前后对照的整体增量 |
| **Uplift / CATE** | 实验天然无混淆;观测需先去混淆 | **个体/子群异质效应** | **targeting:对谁投** |

口径说明(与本仓库其它文档一致):

- **PSM / IPW / AIPW / DID 估的都是平均效应(ATE/ATT)**——回答"这次处理整体带来多少增量"。
- **Uplift 估的是个体/子群异质效应(CATE)**——回答"该把处理投给哪些人"。两者目标不同,不可互相替代。
- **何时用平均效应**:决策是"做不做这个活动""整体值不值",只需一个总量数字。
- **何时用 uplift**:决策是"在预算约束下投给谁""怎么分层施策",需要逐人排序。
- **互补关系**:Uplift 可以**建立在去混淆之后**。观测数据先用 IPW/AIPW 去掉选择偏差,得到无偏的处理-对照比较,再在其上估 CATE——即"先去混淆,再做异质性"。AIPW 的双稳健伪结果也常被直接喂给 uplift 学习器。

## 常见坑

- **用普通分类准确率/AUC/RMSE 评估 uplift**:这些只衡量对结果 $Y$ 的拟合,不衡量对效应 $\tau$ 的排序。一个高 AUC 模型可能把 sure things 排最前,targeting 价值为零。必须用 Qini/uplift 曲线、AUUC。
- **观测数据不去混淆直接建 CATE**:非随机分配时,treated 与 control 本身就不可比,直接作差得到的是"选择偏差 + 处理效应"的混合,uplift 有偏。要先 PSM/IPW/AIPW 去混淆。
- **忽略 sleeping dogs**:只看平均效应为正就全量投放,会漏掉"投了反而流失"的负 uplift 人群,投放不仅浪费还有害。必须解读四象限并主动排除负增益人群。
- **小样本/不平衡上过拟合**:CATE 是两个预测之差,方差天然大;处理组很小、结果稀疏时尤甚。base learner 要正则化、必须在留出集上用排序指标验证,别只看训练集 lift。
- **把 uplift 当响应概率用**:uplift 是"处理 vs 不处理的差",可能为负、绝对值小;按它排序而不是按转化概率排序,才是 targeting 的正确做法。

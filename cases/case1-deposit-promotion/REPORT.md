# 券商入金返券活动因果效应评估报告

> **TL;DR（一句话结论）**：入金返券活动对"注册 → 首次入金"转化率的**真实增量约为 +10 个百分点(pp)**。但若直接对比参加组与未参加组(朴素估计 27.68pp),会**高估约 17 个百分点**——因为本来就更活跃、更优质的用户更倾向于主动报名活动。经 PSM / IPW / AIPW 三种因果推断方法交叉验证,校正后的真实效应稳定落在 **9.40 ~ 11.42 pp** 区间,活动确有可观增量,值得保留并审慎放大。

---

## 1. 业务背景与问题

### 1.1 活动设计
某券商为提升"注册 → 首次入金"的转化,推出返券活动:

> 用户主动**报名/领取活动资格**后,若入金满 **500 美元** 并完成 **3 笔买入交易**,即可获得 **20 美元现金券**。

### 1.2 业务目标
评估该活动对**首次入金转化率**的真实增量贡献,为"是否继续 / 是否扩大投放"提供 ROI 依据。

### 1.3 为什么不能做 A/B 测试
- 活动以"用户主动报名领取资格"的方式触达,**不是随机分配**——是否参加由用户自行决定。
- 由于合规与产品限制,**无法对全量新注册用户做随机 A/B 实验**(不能随机禁止一部分用户领取活动资格)。
- 我们手头只有一张**横截面观测数据快照**:谁报名了、谁没报名、各自是否完成了首次入金,以及他们注册时的可观测特征。

因此,**直接对比"报名组 vs 未报名组"的转化率差是有偏的**——这正是因果推断要解决的核心问题。

---

## 2. 因果框架设定

### 2.1 精确定义(关键设计)

| 要素 | 定义 |
| --- | --- |
| **处理 Treatment (T)** | 用户**是否报名 / 领取了该活动资格**(T=1 报名,T=0 未报名) |
| **结果 Outcome (Y)** | 用户**是否完成首次入金**(Y=1 完成,Y=0 未完成),即首次入金转化 |
| **分析单位** | 单个注册用户 |
| **目标人群** | 实际报名了活动的用户(我们关心活动对**他们**的效应) |
| **估计量 Estimand** | **ATT(Average Treatment effect on the Treated,处理组平均处理效应)** |

> **⚠️ 为什么 Treatment 定义为"是否报名",而不是"是否完成 500 美元 + 3 笔交易"?**
>
> 这是本案例的**核心方法学设计**。"完成入金 500 美元 + 3 笔交易"是活动的**达标条件**,它与结果变量 Y(首次入金)在定义上几乎**同义反复**——一个完成了 500 美元入金的人,几乎必然已"首次入金"。若把它当作 Treatment,就等于用结果解释结果,会造成**病态估计**(估计值人为接近 100%,毫无业务意义)。
>
> 正确做法是把 Treatment 定义在**用户可自主选择、且发生在结果之前**的环节——即"**是否报名领取资格**"。这样我们问的才是一个干净的因果问题:"领取活动资格这一动作,能否提升后续的首次入金转化?"

### 2.2 为什么选 ATT(而非 ATE)
业务问题是"**这次活动对参与者带来了多少增量**",关心的是**实际报名者**这群人,因此估计 ATT 而非全人群 ATE。1:1 最近邻匹配天然以处理组为锚,估计的正是 ATT。

### 2.3 混淆来源分析(自选择偏差)
活跃度高、资金实力强、投资意向强的用户,**同时**满足两个条件:

1. **更可能主动报名活动**(影响 T);
2. **本身就更容易完成首次入金**(影响 Y)。

这类"同时影响 T 和 Y"的因素就是**混淆变量**。它们让报名组的转化率天然高于未报名组——其中一部分是活动真实拉动的,**另一部分只是因为报名的人本来就更优质**。不校正这一偏差,就会把"用户本身优质"误记为"活动有效"。

## 3. 为什么选这些方法

### 3.1 方法路由决策逻辑
依据因果推断方法选择指南(`causal-inference` 路由):

- **第一分叉:要"平均效应"还是"对谁有效"?** → 本案例要的是活动的**平均增量效应**(ATT),不是个体定向,故**排除 Uplift / CATE**。
- **第二分叉:有没有"前后两期 + 对照组"?** → 我们只有**单期横截面快照**,没有上线前后的面板数据,故**排除 DID**(它需要前后期来消除不可观测混淆)。
- **第三分叉:横截面观测数据,在 PSM / IPW / AIPW 中怎么选?** → 三者都校正**可观测混淆**,各有所长,故采用**三方法三角验证**以增强可信度:
  - **PSM**:为每个报名者找一个倾向得分最接近的"双胞胎"未报名者,**可解释性最强**,能向业务方直观展示"可比的处理 vs 未处理个体"。
  - **IPW**:用倾向得分做逆概率加权构造伪总体,**保留全部样本**(不丢弃)。
  - **AIPW(双稳健)**:同时拟合倾向模型 + 结果模型,**只要两者之一设对即一致**,是三者中**稳健性最强**的默认选择。

三种方法基于不同的统计机理,若它们给出一致的结论,则结果的可信度大幅提升(三角验证)。

### 3.2 三种方法各自依赖的关键假设
三种方法共享以下三条**可识别假设**:

1. **可忽略性 / 无混淆(Unconfoundedness / CIA)**:控制了观测协变量 X 后,处理分配与潜在结果独立。通俗说——**所有同时影响"是否报名"和"是否入金"的因素都已被测量并放进模型**。这是最强、也最难验证的假设(数据本身无法证明,只能靠业务知识论证)。
2. **共同支撑 / 正性(Overlap / Positivity)**:每一类用户里,报名组和未报名组都得有人(0 < e(X) < 1),否则没有可比对象。
3. **SUTVA**:个体的结果只取决于自己是否报名,不受他人影响(无溢出),且处理只有单一版本。

> AIPW 额外提供**双稳健性(Double Robustness)**:倾向模型与结果模型**至少一个正确**即保证一致——给了我们"两次机会而非一次"。

## 4. 数据说明

> **⚠️ 重要说明**:本报告使用的是**合成模拟数据**(基于券商业务真实协变量结构生成)。模拟数据的最大价值在于:**真实因果效应(真实 ATT)是已知的**,因此可以用它来**验证因果推断方法是否真的能从被混淆污染的观测数据中还原出真值**。在真实业务数据中,真值永远不可知,这正是为什么我们需要先在模拟环境中验证方法的可靠性。

### 4.1 数据规模
- 总样本量 **n = 10,000** 名注册用户。
- 处理组(报名):**3,425 人(34.2%)**。
- 对照组(未报名):**6,575 人(65.8%)**。

### 4.2 协变量清单(注册时可观测的处理前特征)

| 协变量 | 业务含义 | 与报名/入金的关系 |
| --- | --- | --- |
| `age` | 年龄(标准化) | 年轻用户更可能报名、更易入金 |
| `channel_paid` | 注册渠道(0=自然流量, 1=付费广告) | 付费渠道用户质量更高、更易报名与入金 |
| `device_mobile` | 首次访问设备(0=PC, 1=移动端) | 移动端便利性更高 |
| `activity_days` | 注册后 7 日内 App 登录天数(0~7) | **最强混淆因子**:活跃度越高越可能报名且越易入金 |
| `risk_score` | 风险测评得分(标准化) | 风险偏好高者投资意向更强 |
| `region_tier` | 所在地区分层(1/2/3 线城市) | 低线城市投资意愿更强 |
| `is_bank_linked` | 是否绑定银行卡(0/1) | 绑卡用户入金门槛更低、更易报名与入金 |

### 4.3 如何制造混淆(数据生成机制)
- **报名概率**(Treatment)由上述协变量在 logit 尺度上共同决定:活跃度(系数 +0.8)、绑卡(+0.6)、付费渠道(+0.5)等使用户更可能报名。
- **入金概率**(Outcome)**同时**依赖这些协变量(活跃度 +0.9、绑卡 +0.7 等)**以及报名行为本身**。
- 由于报名与入金**共享同一批驱动因素**,二者天然正相关——这就人为制造了真实的**自选择混淆**,模拟了真实业务中的情形。

### 4.4 真实效应设定
数据生成时,活动的**真实因果效应**被设定为 logit 尺度 +0.50 的增量。在本次运行的样本上,换算到处理组的转化率提升为:

> **真实 ATT = 0.1030(即 +10.30 个百分点转化率提升)**

这是方法需要还原的"标准答案"。

## 5. 分析结果

> 以下所有数字均来自 `analysis.py` 的真实运行输出(随机种子 `RANDOM_SEED = 20260609`,n=10000)。

### 5.1 朴素估计 vs 各方法估计对比

| 方法 | ATT 估计值 | 转化率提升 | 95% 置信区间 | 相对真值偏差 | CI 覆盖真值 |
| --- | --- | --- | --- | --- | --- |
| **真实 ATT(已知真值)** | 0.1030 | **10.30 pp** | — | — | — |
| 朴素估计(未校正) | 0.2768 | 27.68 pp | — | **+17.38 pp(严重高估)** | — |
| **PSM(匹配)** | 0.1142 | 11.42 pp | [0.0894, 0.1391] | +1.11 pp | ✓ |
| **IPW(加权)** | 0.0940 | 9.40 pp | [0.0718, 0.1166] | −0.90 pp | ✓ |
| **AIPW(双稳健)** | 0.0973 | 9.73 pp | [0.0740, 0.1207] | −0.57 pp | ✓ |

**朴素估计的细节**:报名组转化率 52.26%,未报名组转化率 24.58%,直接相减得 27.68pp——这一数字**严重高估了活动真实效果约 17.38pp**,因为它把"报名者本身更优质"也算成了活动功劳。

### 5.2 共同支撑(Overlap)诊断

| 组别 | 倾向得分范围 | 样本量 |
| --- | --- | --- |
| 处理组(报名) | [0.0329, 0.9416] | 3,425 |
| 对照组(未报名) | [0.0144, 0.9294] | 6,575 |
| **共同支撑区间** | **[0.0329, 0.9294]** | — |

两组倾向得分**重叠充分**,positivity 假设大致满足,因果推断有可比基础。

### 5.3 协变量平衡诊断(SMD,目标 |SMD| < 0.1)

匹配前最大失衡来自 `activity_days`(SMD = **0.7422**,即活跃度严重失衡)。**1:1 最近邻匹配 + caliper 后,全部 7 个协变量的 |SMD| 均 < 0.1,达标**:

| 协变量 | 匹配前 SMD | 匹配后 SMD | 是否平衡 |
| --- | --- | --- | --- |
| `age` | −0.3124 | −0.0036 | ✓ |
| `channel_paid` | 0.1977 | −0.0151 | ✓ |
| `device_mobile` | 0.0988 | −0.0032 | ✓ |
| `activity_days` | **0.7422** | −0.0040 | ✓ |
| `risk_score` | 0.1755 | 0.0091 | ✓ |
| `region_tier` | −0.1005 | −0.0212 | ✓ |
| `is_bank_linked` | 0.2868 | 0.0043 | ✓ |

- **匹配率**:3,425 名处理组用户中 **2,794 人成功匹配(81.6%)**,丢弃 631 人(18.4%,因找不到满足 caliper 的可比对照)。
- **caliper** = 0.2 × SD(logit(ps)) = **0.2077**。

> SMD 的意义:它衡量匹配后两组在每个特征上是否真正"可比"。匹配前活跃度等特征严重失衡(报名者明显更活跃),匹配后全部拉平到 0.1 以内,说明我们成功构造了一个**特征分布几乎相同、唯一差别是是否报名**的可比对照——这正是因果推断的前提。

### 5.4 三方法一致性

三种基于不同统计机理的方法,均把朴素估计的 27.68pp **校正回 9.40 ~ 11.42pp**,且**全部 3 个 95% 置信区间都覆盖了真值 10.30pp**:

- PSM:11.42pp(偏差 +1.11pp)
- IPW:9.40pp(偏差 −0.90pp)
- AIPW:9.73pp(偏差 −0.57pp)

三方法收敛于同一结论,且与已知真值高度吻合,**验证了混淆校正的成功**,结果可信度高。

## 6. 业务结论与建议

### 6.1 活动到底有没有效?效果多大?
**有效,且效果可观。** 在剔除"报名者本身更优质"这一自选择偏差后:

> **活动对报名用户的首次入金转化率,带来约 +10 个百分点的真实增量(稳健区间 9.4 ~ 11.4pp)。**

也就是说,在报名的 100 名用户中,大约有 **10 人是因为这个活动才完成首次入金的**(其余的转化即便没有活动也大概率会发生)。

### 6.2 ROI 视角的解读
- **不要用朴素的 27.68pp 来核算 ROI**——那会把活动收益高估近 1.7 倍,导致预算决策严重失真。
- 正确的增量口径是 **~10pp**。据此核算单位获客成本时,应以"每报名 100 人多带来约 10 个首次入金用户"为基准。
- 活动成本(20 美元现金券)只在用户达标(入金 500 美元 + 3 笔交易)时才发放,意味着**成本与高价值转化强绑定**,在 ~10pp 真实增量下,大概率仍具备正向 ROI(具体需结合每位入金用户的 LTV 与实际券核销率测算)。

### 6.3 是否值得继续 / 扩大?
- **建议保留活动**:真实增量稳健为正且幅度可观(~10pp),三方法交叉验证一致。
- **审慎扩大**:当前结论是 **ATT**(对**已报名者**的效应)。若要把活动**推给全量用户**,效应可能不同(全量人群的效应是 ATE,通常因边际用户质量较低而**低于 ATT**)。扩大投放前,建议:
  1. 优先针对**高活跃 / 已绑卡 / 付费渠道**等高响应人群放大(这些群体本身报名与入金倾向最高);
  2. 有条件时,在新一批用户上设计**真随机 A/B 实验**,以获得不受任何假设约束的金标准估计。

## 7. 局限与风险

1. **不可观测混淆(最关键)**:PSM / IPW / AIPW **只能校正我们测量到的混淆**。若存在未纳入模型的因素(如用户固有的投资意愿、外部财力、近期收入变化)同时影响报名与入金,则估计仍可能有偏。这一假设(可忽略性)**数据本身无法证明**,只能靠业务知识论证 + 敏感性分析。
2. **Overlap 风险与样本丢弃**:PSM 因 caliper 限制丢弃了 18.4% 的处理组用户。被丢弃的多是"几乎必然报名、找不到可比对照"的极端用户,结论严格说只适用于**有可比对象的那部分人群**。
3. **模拟数据 vs 真实数据的差距**:本报告基于合成数据(真值已知,用于验证方法可靠性)。真实业务数据中:真值不可知;协变量关系可能非线性、有交互;数据质量(缺失、埋点误差)更复杂。**方法在模拟中验证成功,不等于在真实数据上无偏**——真实落地时务必重新做平衡与 overlap 诊断。
4. **结论的适用边界**:本结论是 **ATT**(对已报名者的平均效应),不能直接外推为"推给全量用户的效应"(ATE),也不能解读为"对每个个体都有 10pp 提升"(那是个体异质效应,需 Uplift/CATE)。
5. **SUTVA**:假设用户之间无溢出(如报名者向好友安利)。若存在社交传播效应,效应估计的解释需谨慎。
